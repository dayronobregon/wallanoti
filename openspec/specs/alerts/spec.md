# Alerts Specification (Duplicate Prevention)\n\n## Purpose\n\nThis specification defines the complete behavior for processing Wallapop items during alert searches, including persistence of ProcessedItems and detection of new items, updates (ModifiedAt changes), and price drops to avoid duplicate Telegram notifications.\n\n## Requirements\n\n### Requirement: ProcessedItem Persistence\n\nThe system **SHALL** persist a `ProcessedItem` record for each unique `AlertId` + `Wallapop ItemId` combination. Each record **SHALL** store `LastModifiedAt` (from Wallapop `ModifiedAt`) and `LastPrice` (from Wallapop `CurrentPrice`).\n\nThe record **SHALL** be updated after every fetch with the latest values.\n\n#### Scenario: Persist new ProcessedItem (Happy Path)\n\n- GIVEN an active alert and a newly fetched Wallapop item with no existing ProcessedItem\n- WHEN `ItemSearcher` processes the item\n- THEN create and persist ProcessedItem with current `ModifiedAt` and `CurrentPrice`\n- AND publish `NewItemsFoundEvent` with `changeType: \"new\"`\n\n#### Escenario: Persistir nuevo ProcessedItem (Happy Path - Español)\n\n- SI un alerta activa y un ítem de Wallapop recién fetched sin ProcessedItem existente\n- CUANDO `ItemSearcher` procesa el ítem\n- ENTONCES crear y persistir ProcessedItem con `ModifiedAt` y `CurrentPrice` actuales\n- Y publicar `NewItemsFoundEvent` con `changeType: \"new\"`\n\n### Requirement: New Item Detection\n\nThe system **SHALL** classify an item as **new** when no ProcessedItem record exists for the alert-item pair.\n\n#### Scenario: New item notification\n\n- GIVEN no ProcessedItem for alert A and item I\n- WHEN item I is fetched (CreatedAt >= alert.CreatedAt)\n- THEN `changeType = \"new\"`\n- AND notification sent with \"Nuevo:\" prefix\n\n### Requirement: Item Update Detection\n\nThe system **SHALL** detect an **update** when `item.ModifiedAt > processed.LastModifiedAt`.\n\n#### Scenario: Update without price change\n\n- GIVEN ProcessedItem with LastModifiedAt = \"2024-01-01T10:00\", LastPrice = 100€\n- WHEN fetched item has ModifiedAt = \"2024-01-01T11:00\", CurrentPrice = 100€\n- THEN `changeType = \"update\"`\n- AND update ProcessedItem record\n- AND notification with \"Actualización:\" prefix\n\n#### Escenario: Actualización sin cambio de precio (Español)\n\n- DADO ProcessedItem con LastModifiedAt = \"2024-01-01T10:00\", LastPrice = 100€\n- CUANDO ítem fetched tiene ModifiedAt = \"2024-01-01T11:00\", CurrentPrice = 100€\n- ENTONCES `changeType = \"update\"`\n- Y actualizar ProcessedItem\n- Y notificación con \"Actualización:\" prefijo\n\n### Requirement: Price Drop Detection\n\nThe system **SHALL** detect a **price drop** when `item.CurrentPrice < processed.LastPrice`, independently of ModifiedAt.\n\n#### Scenario: Price drop (no ModifiedAt change)\n\n- GIVEN LastPrice = 100€, LastModifiedAt any\n- WHEN CurrentPrice = 90€\n- THEN `changeType = \"price_drop\"`\n- AND update record with new price\n- AND notification with \"Bajada de Precio:\" prefix\n\n#### Scenario: Price drop with ModifiedAt change\n\n- GIVEN LastPrice = 100€, ModifiedAt old\n- WHEN CurrentPrice = 90€ AND newer ModifiedAt\n- THEN `changeType = \"price_drop\"` (priority over update)\n- AND notification \"Bajada de Precio:\"\n\n### Requirement: No Notification on No Change\n\nThe system **SHALL NOT** send notification if `ModifiedAt <= LastModifiedAt` AND `CurrentPrice >= LastPrice`.\n\n#### Scenario: No changes\n\n- GIVEN LastModifiedAt = \"2024-01-01T12:00\", LastPrice = 100€\n- WHEN item has ModifiedAt = \"2024-01-01T11:00\", CurrentPrice = 100€\n- THEN no event published\n- AND update ProcessedItem if fetched (but no notify)\n\n## Edge Cases\n\n### Scenario: False ModifiedAt (same ModifiedAt, price drop)\n\n- GIVEN LastModifiedAt = \"2024-01-01T10:00\", LastPrice = 100€\n- WHEN ModifiedAt = \"2024-01-01T10:00\" (false/no change), CurrentPrice = 90€\n- THEN detect price drop, notify \"Bajada de Precio:\"\n- AND update LastPrice to 90€\n\n### Scenario: Multi-alerts for same item\n\n- GIVEN two alerts A1, A2 for same search\n- WHEN item I updates\n- THEN A1 ProcessedItem updated and notifies for A1\n- AND A2 ProcessedItem updated and notifies for A2 (independent)\n\n### Scenario: Price increase\n\n- GIVEN LastPrice = 100€\n- WHEN CurrentPrice = 110€\n- THEN no notification (only drops)\n- AND update LastPrice to 110€

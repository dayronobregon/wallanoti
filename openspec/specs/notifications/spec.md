# Notifications Specification (Prefixes for Changes)\n\n## Purpose\n\nThis specification defines how notifications are formatted with Spanish prefixes indicating the type of change detected in Wallapop items.\n\n## Requirements\n\n### Requirement: Change Type Prefixes\n\nThe system **SHALL** prefix the notification title with one of the following based on `changeType` from `NewItemsFoundEvent`:\n\n- `changeType == \"new\"`: \"**Nuevo:** \"\n- `changeType == \"update\"`: \"**Actualización:** \"\n- `changeType == \"price_drop\"`: \"**Bajada de Precio:** \"\n\n#### Scenario: New item prefix (English)\n\n- GIVEN `changeType = \"new\"`\n- WHEN `Notification.FormattedString()` is called\n- THEN title starts with \"**Nuevo:** \"\n\n#### Escenario: Prefijo para nuevo ítem (Español)\n\n- SI `changeType = \"new\"`\n- CUANDO `Notification.FormattedString()` es llamada\n- ENTONCES título comienza con \"**Nuevo:** \"\n\n#### Scenario: Update prefix\n\n- GIVEN `changeType = \"update\"`\n- WHEN formatting\n- THEN \"**Actualización:** \"\n\n#### Escenario: Prefijo de actualización\n\n- SI `changeType = \"update\"`\n- CUANDO formateando\n- ENTONCES \"**Actualización:** \"\n\n#### Scenario: Price drop prefix\n\n- GIVEN `changeType = \"price_drop\"`\n- WHEN formatting\n- THEN \"**Bajada de Precio:** \"\n\n#### Escenario: Prefijo bajada de precio\n\n- SI `changeType = \"price_drop\"`\n- CUANDO formateando\n- ENTONCES \"**Bajada de Precio:** \"\n\n### Requirement: Priority for Multiple Changes\n\nIf both update and price drop conditions met, **SHALL** use \"price_drop\" prefix.\n\n#### Scenario: Priority price drop\n\n- GIVEN conditions for both update and price_drop\n- WHEN determining prefix\n- THEN use \"**Bajada de Precio:** \"
